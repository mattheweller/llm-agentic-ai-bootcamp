Task 4

Format Data for Supervised Fine-Tuning

Learning Objectives

  • Understand why instruction-tuned models need a specific prompt format
  • Learn how apply_chat_template() wraps messages in model-specific tokens
  • See the difference in templates between Gemma and DeepSeek models
  • Format the full dataset using a mapping function for SFTTrainer

Why Format Matters

Instruction-tuned models (like gemma-3-1b-it) were trained on conversations structured with special tokens that mark turn boundaries. When you fine-tune, your training examples must match the exact same format the model was pre-trained on — otherwise the model gets confused about when it's the user's turn vs. its own turn to respond.

The apply_chat_template() tokenizer method handles this automatically. You pass a list of {"role": ..., "content": ...} dicts, and it serialises them into the model's expected string format.

Gemma's Chat Template

Gemma uses four special tokens to delineate conversation turns:

TokenMeaning
<bos>Beginning of sequence — tells the model to start reading
<start_of_turn>userStart of the human's turn
<end_of_turn>End of the current speaker's turn
<start_of_turn>modelStart of the model's response

The Formatting Function

Each training example becomes a full conversation: the user provides the instruction + headline, and the model's expected response is the sentiment label.

Python — format_for_sft_gemma() 
def format_for_sft_gemma(example, tokenizer):
    system_prompt = (
        "Classify the sentiment of the following sentence from Financial News. "
        "Respond with ONLY ONE of the following labels: positive, negative, neutral."
    )
    messages = [
        {"role": "user",  "content": f"{system_prompt}\n\nSentence: {example['text']}"},
        {"role": "model", "content": example["sentiment"]},
    ]
    # add_generation_prompt=False because we include the answer in training data
    formatted = tokenizer.apply_chat_template(
        messages, tokenize=False, add_generation_prompt=False
    )
    return {"text": formatted}

# Apply to the full dataset
sft_train_dataset = train_dataset.map(
    format_for_sft_gemma,
    fn_kwargs={"tokenizer": base_os_tokenizer},
    remove_columns=list(train_dataset.features),
)

What SFTTrainer Does With This

The SFTTrainer uses causal language modelling: given the formatted string, it trains the model to predict the next token at every position. But here's the key — by default it only computes loss on the completion tokens (the label), not the prompt tokens. This prevents the model from wasting capacity memorising the instruction.

Input tokens (no loss)

<bos><start_of_turn>user
Classify the sentiment...
Sentence: Operating profit...
<end_of_turn>
<start_of_turn>model

Completion tokens (loss computed here)

positive<end_of_turn>

The model is trained to predict "positive" after seeing the full instruction and headline.

Practice: Change the model in the formatter to DeepSeek-R1-Distill. Notice how the special tokens change completely — <｜User｜>, <｜Assistant｜>, etc. Each model family has its own template. This is why you always use apply_chat_template() rather than manually constructing the string.
