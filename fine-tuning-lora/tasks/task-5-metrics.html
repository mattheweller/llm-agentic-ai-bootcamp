Task 5

Confusion Matrix & Classification Metrics

Learning Objectives

  • Understand why accuracy alone is misleading for imbalanced datasets
  • Learn precision, recall, and F1-score with intuitive definitions
  • Read and interpret a multi-class confusion matrix
  • Use an interactive calculator to see how metrics relate to raw counts

Why Not Just Use Accuracy?

Our dataset is ~60% neutral. A model that predicts "neutral" for everything achieves 60% accuracy — but it's completely useless. Accuracy hides this because it treats every sample equally regardless of class.

We need metrics that separately evaluate performance per class and surface the model's handling of minority classes.

The Building Blocks: TP, FP, FN, TN

For each class (e.g., "positive"), every prediction falls into one of four categories:

Predicted Positive Predicted NOT Positive
Actually Positive TP — True Positive (correct!) FN — False Negative (missed it)
Actually NOT Positive FP — False Positive (wrong alarm) TN — True Negative (correct!)

Precision, Recall, and F1

Precision

TP / (TP + FP)

Of everything the model called "positive", what fraction actually was? Low precision = too many false alarms.

Recall

TP / (TP + FN)

Of all actual "positive" samples, what fraction did the model catch? Low recall = too many misses.

F1-Score

2 × (P × R) / (P + R)

Harmonic mean of precision and recall. Punishes extreme imbalance between the two — the go-to metric for classification.

Multi-Class Confusion Matrix

For three classes (positive, negative, neutral), the confusion matrix becomes a 3×3 grid. The diagonal cells (green) are correct predictions; off-diagonal cells are errors.

How to read it: Each row represents the actual class. Each column represents the predicted class. Row totals = true class counts. The diagonal value / row total = per-class recall.

Example (baseline zero-shot predictions on 485 test samples):

Predicted →

actual: neutral 130 100 60 290

actual: negative 20 12 28 60

Reading the "positive" row: of 135 actual positive samples, the model correctly labelled 55 (recall = 41%), misclassified 55 as neutral, and 25 as negative. Poor recall on positive class is typical for zero-shot financial sentiment.

The sklearn Code

Python — Compute Metrics 
from sklearn.metrics import accuracy_score, classification_report, confusion_matrix

labels = ["positive", "negative", "neutral"]

accuracy = accuracy_score(true_labels, predictions)
report   = classification_report(
    true_labels, predictions,
    labels=labels, zero_division=0, target_names=labels
)
cm = confusion_matrix(true_labels, predictions, labels=labels)

print(f"Accuracy: {accuracy:.4f}")
print(report)
