Libraries & GPU Setup

Learning Objectives

  • Understand why GPU is required for fine-tuning (and why CPU won't work)
  • Learn the role of each library in the Hugging Face ecosystem
  • Set up Hugging Face Hub authentication for gated models
  • Verify GPU availability before starting training
GPU Required — This Won't Run Locally on macOS/Windows
The bitsandbytes quantization library requires a CUDA GPU (NVIDIA). Use Google Colab with a T4 or better runtime. Free tier is sufficient.

The Hugging Face Ecosystem

Fine-tuning a modern LLM uses five distinct libraries that each handle one layer of the stack. It's worth understanding what each one does before writing code.

transformers
Model architectures, tokenizers, and the base training loop (Trainer API)
datasets
Download, cache, and stream datasets from Hugging Face Hub with lazy loading
peft
Parameter-Efficient Fine-Tuning — wraps any model with LoRA, QLoRA, adapters
trl
SFTTrainer, reward models, RLHF/DPO training — simplifies the fine-tuning loop
bitsandbytes
4-bit and 8-bit quantization — loads a 1B model in ~700MB VRAM instead of ~2GB
accelerate
Device management — automatically places model layers on GPU/CPU as needed

Hugging Face Hub Authentication

Gemma 3 is a gated model — you must accept Google's terms on the model card before you can download it. Two steps:

  1. Go to the model page on Hugging Face and accept the terms of use (search for google/gemma-3-1b-it)
  2. Generate a read token in your HF account settings and authenticate in Colab:
Python — Cell 2 
from huggingface_hub import notebook_login
notebook_login()  # Opens a widget to paste your HF token

GPU Detection

Always verify GPU availability before starting. Training silently falls back to CPU if CUDA isn't found, and will appear to run — extremely slowly — until it times out.

Python — Cell 3 
import torch

if torch.cuda.is_available():
    device = torch.device("cuda")
    gpu_name = torch.cuda.get_device_name(0)
    vram_gb = torch.cuda.get_device_properties(0).total_memory / 1e9
    print(f"GPU detected: {gpu_name}")
    print(f"VRAM: {vram_gb:.1f} GB")
else:
    print("No GPU detected — enable GPU runtime before continuing!")
    print("Runtime > Change runtime type > T4 GPU")

Practice: After running the GPU check cell, look at your VRAM figure. A T4 has ~15 GB. A 1B parameter model in float32 needs ~4 GB — but with 4-bit quantization it fits in under 1 GB, leaving room for gradients and activations during training. That's why QLoRA exists.
