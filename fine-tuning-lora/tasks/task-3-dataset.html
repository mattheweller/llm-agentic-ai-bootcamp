Task 3

Load & Explore the Dataset

Learning Objectives

  • Load a dataset from Hugging Face Hub using the datasets library
  • Inspect dataset structure, features, and label distribution
  • Understand why class imbalance matters for classification metrics
  • Create a reproducible train/test split

The Dataset

We use the Daniel-ML/sentiment-analysis-for-financial-news-v2 dataset from Hugging Face Hub. It contains 4,847 labelled financial news sentences — real headlines and article fragments tagged by financial analysts.

Python — Load Dataset 
from datasets import load_dataset

dataset_id = "Daniel-ML/sentiment-analysis-for-financial-news-v2"
labeled_dataset = load_dataset(dataset_id, split="train")

print(labeled_dataset)

Class Distribution

Financial news datasets are typically imbalanced — most headlines are factual/neutral statements. Understanding this distribution before training is critical.

Class Imbalance Matters: A naive model could achieve ~60% accuracy by predicting "neutral" for everything. This is why we use F1-score (which penalises poor minority-class performance) rather than raw accuracy as our primary metric.

Splitting the Data

We use a 90/10 train/test split with a fixed seed for reproducibility. The datasets library handles this natively:

Python — Train/Test Split 
split_dataset = labeled_dataset.train_test_split(
    test_size=0.10,
    seed=42,
    shuffle=True,
)
train_dataset = split_dataset["train"]   # 4,362 samples
test_dataset  = split_dataset["test"]    #   485 samples

print(f"Training: {len(train_dataset)} samples")
print(f"Testing:  {len(test_dataset)} samples")
