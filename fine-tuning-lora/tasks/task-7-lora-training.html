Task 7

Fine-Tuning with PEFT & LoRA

Learning Objectives

  • Understand the LoRA rank decomposition — the math behind why it works
  • Configure LoRA hyperparameters and understand the tradeoffs
  • Use the interactive parameter counter to see how few weights LoRA actually trains
  • Run the SFTTrainer training loop and interpret the loss curve

The Problem with Full Fine-Tuning

Full fine-tuning updates every weight in the model. For Gemma 3 1B, that's ~1 billion parameters. In float32, storing gradients and optimizer states (Adam needs 2 copies of every gradient) requires:

1B params × (4 bytes model + 4 bytes gradient + 8 bytes Adam) = ~16 GB

That exceeds a T4 GPU's 15 GB VRAM. Full fine-tuning of a 1B model requires a V100 or A100.

LoRA: Low-Rank Adaptation

LoRA (Hu et al., 2021) solves this by making a key observation: the weight updates needed for task adaptation have low intrinsic rank. Instead of updating the full weight matrix W, LoRA decomposes the update into two small matrices:

W' = W + ΔW = W + B·A
where W ∈ ℝd×k, B ∈ ℝd×r, A ∈ ℝr×k, and r << min(d, k)

At inference, B·A is merged into W — no latency penalty. During training, only B and A are updated, with B initialised to zeros so the adapter starts as a no-op.

Training Arguments

Python — TrainingArguments 
training_args = TrainingArguments(
    output_dir="./sentiment_finetuned_adapter",
    per_device_train_batch_size=4,   # Increase if VRAM allows
    gradient_accumulation_steps=2,   # Effective batch = 4 × 2 = 8
    learning_rate=2e-4,              # Higher than full fine-tuning (LoRA adapters are small)
    num_train_epochs=4,
    lr_scheduler_type="cosine",
    warmup_ratio=0.05,
    fp16=True,                       # Half precision — saves VRAM, speeds training
    logging_steps=20,
    save_strategy="epoch",
    report_to="none",
)

Launching the SFTTrainer

Python — SFTTrainer 
trainer = SFTTrainer(
    model=base_os_model,
    train_dataset=sft_train_dataset,
    peft_config=lora_config,
    args=training_args,
    tokenizer=base_os_tokenizer,
)

trainer.train()
trainer.save_model()  # Saves only the LoRA adapter weights (~4MB)

What Gets Saved?

SFTTrainer saves only the LoRA adapter weights, not the full model. The adapter directory contains:

Adapter Directory
sentiment_finetuned_adapter/
├── adapter_config.json      # LoRA config (r, alpha, target_modules)
├── adapter_model.safetensors  # The trained B and A matrices (~4 MB)
├── tokenizer.json
├── tokenizer_config.json
└── special_tokens_map.json

To deploy, you load the base model (via HF Hub) and merge the adapter. The 4MB adapter embodies everything the model learned about financial sentiment.

Key Insight: LoRA trains 0.23% of the model's parameters, takes ~15 minutes on a T4, produces a 4MB adapter file, and delivers a jump from 44% to 88% accuracy on this task. That's the efficiency argument for PEFT in production.
