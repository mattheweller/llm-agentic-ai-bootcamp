Task 6

Zero-Shot Baseline Evaluation

Learning Objectives

  • Understand what "zero-shot" means and why it's the correct baseline
  • Load a quantized model and run inference on the full test set
  • Interpret the baseline confusion matrix and per-class F1 scores
  • Identify which classes the base model struggles with most

What Is Zero-Shot Classification?

Zero-shot means running the model with only a prompt — no examples, no fine-tuning, no training at all. We describe the task in the system prompt and ask the model to classify. This establishes a fair baseline: what can a general-purpose 1B model already do on this specific task?

Why measure the baseline first? Fine-tuning is expensive and time-consuming. If the baseline already achieves 85%+ F1, fine-tuning may not be worth the cost. The baseline also reveals which classes need the most work.

The Inference Pipeline

Python — Zero-Shot Inference Function 
def classify_zero_shot(sentence, model, tokenizer):
    system_prompt = (
        "Classify the sentiment of the following sentence from Financial News. "
        "Respond with ONLY ONE of the following labels: positive, negative, neutral."
    )
    messages = [
        {"role": "user", "content": f"{system_prompt}\n\nSentence: {sentence}"}
    ]
    prompt = tokenizer.apply_chat_template(
        messages, tokenize=False, add_generation_prompt=True
    )
    inputs = tokenizer(prompt, return_tensors="pt").to(model.device)

    with torch.no_grad():
        output_ids = model.generate(
            **inputs,
            max_new_tokens=10,   # Only need 1-2 tokens for the label
            do_sample=False,     # Greedy decoding — deterministic
            pad_token_id=tokenizer.eos_token_id,
        )
    # Decode only the newly generated tokens (not the prompt)
    new_tokens = output_ids[0][inputs["input_ids"].shape[1]:]
    response   = tokenizer.decode(new_tokens, skip_special_tokens=True).strip().lower()

    if response in ["positive", "negative", "neutral"]:
        return response
    return "unknown"

Reading the Results

  • Neutral confusion: The model misclassifies 160 of 290 neutral headlines (~55%) — neutral financial language is too subtle for zero-shot prompting.
  • Positive/negative confusion: 25 actual-negative headlines predicted as positive — the base model conflates factual announcements with positive sentiment.
  • Overall F1 of 0.38 is the score to beat. After LoRA fine-tuning this rises to ~0.88.
Practice: In the notebook, try modifying the system_prompt — remove the label list, simplify the instruction, or add one-shot examples. How much does prompting engineering improve the baseline? This is a good sanity check before investing in fine-tuning.
