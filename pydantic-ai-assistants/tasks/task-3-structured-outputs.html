Task 3

Structured LLM Outputs with JSON Schema

Learning Objectives

  • Understand OpenAI's response_format parameter
  • Convert Zod schemas to JSON Schema for LLM validation
  • Guarantee schema compliance from LLM responses
  • Handle validation errors with fallback strategies

The Problem: Unreliable JSON from LLMs

When you ask an LLM to return JSON without constraints, you get unpredictable results:

Common Issues:
  • Missing required fields
  • Wrong data types (strings instead of numbers)
  • Extra fields you didn't ask for
  • Malformed JSON with syntax errors

The Solution: OpenAI Structured Outputs

OpenAI's response_format parameter with json_schema enforces strict schema compliance. The model guarantees valid output that matches your schema:

Backend Implementation
import { z } from 'zod';
import { zodToJsonSchema } from 'zod-to-json-schema';
import OpenAI from 'openai';

// Define Zod schema
const RecipeSchema = z.object({
  name: z.string(),
  cuisine: z.string(),
  difficulty: z.enum(['easy', 'medium', 'hard']),
  prep_time_minutes: z.number().int().positive(),
  ingredients: z.array(z.string()).min(3),
  instructions: z.array(z.string()).min(3)
});

// Convert to JSON Schema
const jsonSchema = zodToJsonSchema(RecipeSchema, 'recipe');

// Call OpenAI with structured output
const response = await openai.chat.completions.create({
  model: 'gpt-4o',
  messages: [
    { role: 'system', content: 'You are a helpful cooking assistant.' },
    { role: 'user', content: 'Give me a recipe for chocolate chip cookies' }
  ],
  response_format: {
    type: 'json_schema',
    json_schema: {
      name: 'recipe',
      strict: true,
      schema: jsonSchema
    }
  }
});

// Parse and validate
const content = JSON.parse(response.choices[0].message.content);
const validated = RecipeSchema.parse(content);
// ✅ Guaranteed to match schema!

Schema Validation Flow

Here's how structured outputs work end-to-end:

  1. Define Zod Schema: Specify exact structure with types and constraints
  2. Convert to JSON Schema: Use zodToJsonSchema() for OpenAI compatibility
  3. Call OpenAI: Pass JSON Schema in response_format parameter
  4. Validate Response: Parse and verify with Zod (should never fail if schema is correct)
Without Structured Outputs
// ❌ No guarantees
const response = await openai.chat.completions.create({
  model: 'gpt-4o',
  messages: [
    { role: 'user', content: 'Return recipe as JSON' }
  ]
});

// Could be anything - validation might fail
const data = JSON.parse(response.choices[0].message.content);
const validated = RecipeSchema.parse(data); // Might throw!
With Structured Outputs
// ✅ Guaranteed compliance
const response = await openai.chat.completions.create({
  model: 'gpt-4o',
  messages: [
    { role: 'user', content: 'Give me a recipe' }
  ],
  response_format: {
    type: 'json_schema',
    json_schema: { name: 'recipe', strict: true, schema: jsonSchema }
  }
});

// Always valid - no validation errors
const data = JSON.parse(response.choices[0].message.content);
const validated = RecipeSchema.parse(data); // Always succeeds!

Key Concepts

Concept Description
response_format OpenAI parameter that enforces output structure
json_schema Standard format for defining JSON structure (not Zod-specific)
strict: true Enables strict mode - model must follow schema exactly
zodToJsonSchema() Converts Zod schema to JSON Schema for OpenAI
schema.parse() Validates data against Zod schema (throws on failure)

Benefits for Production Systems

  • ✅ No Runtime Errors: LLM output guaranteed to match expected structure
  • ✅ Type Safety: Frontend and backend agree on data shape
  • ✅ Reduced Costs: No retry loops for malformed JSON
  • ✅ Predictable Behavior: Same schema every time, no surprises
When to Use: Use structured outputs whenever you need predictable data from LLMs - API responses, database writes, multi-step pipelines, or any production system.
