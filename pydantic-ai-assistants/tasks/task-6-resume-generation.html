Task 6

Generate Tailored Resume with Diff Highlighting

Learning Objectives

  • Generate an improved resume tailored to job requirements
  • Use structured outputs to return both resume text and diff HTML
  • Display changes with color-coded highlighting (additions in green, deletions in red)
  • Understand how to chain LLM outputs through multi-step workflows

Resume Generation Schema

The LLM returns both the updated resume and an HTML diff showing changes:

Backend Schema
const ResumeOutputSchema = z.object({
  updated_resume: z.string().min(100),
  diff_html: z.string()
});

// Example response structure:
{
  updated_resume: "**Alex Chen**\n...",  // Full new resume text
  diff_html: "
Added text...
"
}

How It Works

The resume generation follows a three-step process:

  1. Load Context: Retrieve resume, job description, and gap analysis from localStorage
  2. Call LLM: Send all context to OpenAI with structured output schema
  3. Display Results: Show updated resume and visual diff side-by-side
Frontend Implementation
async function generateResume() {
  // Load context from previous tasks
  const resume = localStorage.getItem('pydantic_resume');
  const jobDesc = localStorage.getItem('pydantic_jobdesc');
  const gapAnalysis = JSON.parse(
    localStorage.getItem('pydantic_gap_analysis')
  );

  // Call backend endpoint
  const response = await fetch('/api/structured/generate-resume', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      resume_text: resume,
      job_description_text: jobDesc,
      gap_analysis: gapAnalysis
    })
  });

  const result = await response.json();
  // result.updated_resume - new resume text
  // result.diff_html - color-coded changes

  displayResume(result);
}

LLM Prompt Strategy

The backend sends detailed instructions to ensure high-quality output:

Prompt Guidelines:
  • Rewrite resume to match job description keywords (ATS optimization)
  • Address gaps identified in the analysis
  • Quantify achievements with metrics where possible
  • Use strong action verbs (developed, implemented, optimized)
  • Generate inline HTML diff for visual comparison

Key Features

  • ✅ ATS-Optimized: Keywords from job description integrated naturally
  • ✅ Gap Awareness: Addresses weaknesses identified in Task 5
  • ✅ Visual Diff: See exactly what changed with color highlighting
  • ✅ Copy/Download: Export updated resume for immediate use
