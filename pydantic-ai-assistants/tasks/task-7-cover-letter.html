Tone-Aware Cover Letter Generation

Learning Objectives

  • Generate cover letters with tone control (professional, enthusiastic, startup-friendly)
  • Use structured outputs to guarantee format compliance
  • Implement copy-to-clipboard and download functionality
  • Understand how tone affects LLM writing style

Cover Letter Schema

The schema enforces length constraints to ensure concise, focused cover letters:

Backend Schema
const CoverLetterSchema = z.object({
  cover_letter: z.string().min(200).max(2000)
});

// Validation guarantees:
// ✅ At least 200 characters (filters out too-short responses)
// ✅ Maximum 2000 characters (prevents overly verbose letters)
// ✅ Returns only the letter text, no metadata

How Tone Affects Output

The backend maps tone values to specific prompt guidance:

Tone Guidance Map
const toneGuidance = {
  professional:
    'formal and corporate-appropriate, emphasizing qualifications and experience',
  enthusiastic:
    'energetic and passionate while remaining professional, showing genuine interest',
  'startup-friendly':
    'slightly informal but polished, emphasizing adaptability and innovation'
};

// Injected into prompt:
`Tone: ${tone} (${toneGuidance[tone]})`

Cover Letter Requirements

All generated cover letters follow these guidelines:

  • ✅ Generic greeting (e.g., "Dear Hiring Manager")
  • ✅ 3-4 paragraphs maximum
  • ✅ Highlights key achievements from updated resume
  • ✅ Aligns with job description requirements
  • ✅ Expresses enthusiasm and fit for the role
  • ✅ Confident and polite closing statement
  • ✅ Matches selected tone throughout

Typical Cover Letter Structure

Section Purpose
Opening Paragraph State the position, express interest, mention where you found the job
Body Paragraph 1 Highlight relevant experience and key achievements from resume
Body Paragraph 2 Explain why you're a great fit and what you bring to the role
Closing Paragraph Express enthusiasm, mention next steps, thank the reader
Pro Tip: Try generating with different tones to see which matches the company culture. Startups often respond better to enthusiastic or startup-friendly tones, while enterprises prefer professional.
