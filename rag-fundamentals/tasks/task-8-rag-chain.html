The RAG Pipeline

RAG combines two steps:

  1. Retrieve: Find relevant chunks using semantic search
  2. Generate: Feed chunks + query to LLM for answer

💡 Why RAG Works: The LLM gets specific, relevant context instead of relying on training data. This reduces hallucinations and provides up-to-date, verifiable answers.

Code Example: RAG Chain

async function ragChain(question) {
  // Step 1: Retrieve relevant chunks
  const queryEmbedding = await apiClient.embed(question, 'openai');

  const results = vectorStore.map(item => ({
    ...item,
    similarity: cosineSimilarity(queryEmbedding, item.embedding)
  }));

  results.sort((a, b) => b.similarity - a.similarity);
  const topChunks = results.slice(0, 3);

  // Step 2: Build context from retrieved chunks
  const context = topChunks.map(r => r.text).join('\n\n');

  // Step 3: Create prompt with context
  const prompt = `You are a helpful assistant for Eleven Madison Park restaurant.

Use the following context to answer the question. If you cannot answer based on the context, say so.

Context:
${context}

Question: ${question}

Answer:`;

  // Step 4: Generate answer
  const answer = await apiClient.simpleChat(prompt, 'openai', {
    temperature: 0.3,  // Lower temp for factual answers
    max_tokens: 300
  });

  return { answer, context, retrievedChunks: topChunks };
}

Prompt Engineering Tips

  • Clear role: "You are a helpful assistant for..."
  • Explicit instructions: "Use the context to answer..."
  • Honesty directive: "If you cannot answer, say so"
  • Structure: Context → Question → Answer
  • Lower temperature: 0.2-0.4 for factual accuracy
