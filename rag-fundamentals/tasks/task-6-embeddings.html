What Are Embeddings?

Embeddings are numerical representations of text that capture semantic meaning:

  • High-dimensional vectors (e.g., 1536 dimensions for OpenAI's text-embedding-3-small)
  • Similar meanings = similar vectors in vector space
  • Enable semantic search - find related content, not just keyword matches
  • Foundation for RAG - allows retrieving relevant chunks by meaning

💡 Key Insight: "dog" and "puppy" have different letters but similar embeddings because they have similar meanings. Traditional keyword search misses this!

Code Example: Generate Embeddings

// Generate embeddings for all chunks
const apiClient = new LLMAPIClient('http://localhost:3000');

const vectorStore = [];

for (let i = 0; i < chunks.length; i++) {
  const chunk = chunks[i];

  // Generate embedding for this chunk
  const embedding = await apiClient.embed(chunk, 'openai');

  // Store with metadata
  vectorStore.push({
    id: i,
    text: chunk,
    embedding: embedding,  // Array of 1536 numbers
    metadata: {
      chunkIndex: i,
      length: chunk.length
    }
  });

  console.log(`Embedded chunk ${i + 1}/${chunks.length}`);
}

console.log('Vector store ready!', vectorStore);

Understanding the Vector Store

Our simple in-memory vector store contains:

  • Text: The original chunk content
  • Embedding: 1536-dimensional vector representing the chunk's meaning
  • Metadata: Additional info (index, length, etc.)
  • ID: Unique identifier for retrieval

💡 In Production: You'd use a specialized vector database like Pinecone, Weaviate, or ChromaDB. They're optimized for fast similarity search at scale.
