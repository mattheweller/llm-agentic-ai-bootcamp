Document Loading & Chunking

Learning Objectives

  • Load and parse text documents from files
  • Understand why chunking is necessary for RAG
  • Learn different chunking strategies
  • Implement recursive character-based text splitting

Why Chunking Matters

LLMs have context windows (token limits). For effective RAG:

  • Documents are too large to fit in the context window
  • We need semantic units that contain complete thoughts
  • Smaller chunks improve retrieval precision - more focused matches
  • Balance is key - too small loses context, too large loses specificity

💡 Key Insight: Good chunking preserves semantic meaning while staying within token limits. A chunk should contain enough context to be meaningful on its own.

Chunking Strategies

Common approaches to split documents:

Strategy Pros Cons
Fixed-size Simple, predictable May break mid-sentence
Sentence-based Preserves sentences Variable chunk sizes
Paragraph-based Natural boundaries Can be too large
Recursive (recommended) Tries paragraphs, then sentences, then characters Slightly more complex

Code Example: Recursive Text Splitter

// Simple recursive text splitter
function splitTextRecursive(text, chunkSize, chunkOverlap) {
  const separators = ['\n\n', '\n', '. ', ' ', ''];
  return splitBySeparators(text, separators, chunkSize, chunkOverlap);
}

function splitBySeparators(text, separators, chunkSize, overlap) {
  if (!text || text.length <= chunkSize) return [text];

  const [sep, ...restSeps] = separators;
  const chunks = [];
  const parts = text.split(sep);

  let currentChunk = '';

  for (const part of parts) {
    if ((currentChunk + sep + part).length <= chunkSize) {
      currentChunk += (currentChunk ? sep : '') + part;
    } else {
      if (currentChunk) chunks.push(currentChunk);

      // If part is still too large, recurse with next separator
      if (part.length > chunkSize && restSeps.length > 0) {
        chunks.push(...splitBySeparators(part, restSeps, chunkSize, overlap));
      } else {
        currentChunk = part;
      }
    }
  }

  if (currentChunk) chunks.push(currentChunk);
  return chunks;
}

Key Takeaways

  • Chunking is essential for fitting documents into LLM context windows
  • Recursive splitting tries to preserve semantic boundaries (paragraphs → sentences → characters)
  • Overlap prevents information loss at chunk boundaries
  • Chunk size is a tradeoff between context and precision
