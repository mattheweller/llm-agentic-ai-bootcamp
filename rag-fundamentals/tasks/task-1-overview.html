Project: Ask Eleven Madison Park Restaurant

In this module, you'll build an intelligent Q&A assistant for Eleven Madison Park, a renowned fine dining restaurant in New York City. Your assistant will answer questions about:

  • Menu options and dietary accommodations
  • Reservation policies and procedures
  • Dress code requirements
  • Team members and restaurant philosophy
  • Hours, location, and contact information

The Challenge

Traditional chatbots struggle with specific, domain-focused questions because:

  • Limited Knowledge: LLMs are trained on general data but lack specific, up-to-date information about individual restaurants
  • Hallucinations: Without grounding in real data, LLMs may make up plausible-sounding but incorrect answers
  • No Sources: Users can't verify where information comes from
Real Problem: Asking GPT-4 "What menus does Eleven Madison Park offer?" might produce outdated or generic information. Our RAG system will answer from actual restaurant data.

The Solution: RAG (Retrieval Augmented Generation)

RAG solves these problems by:

  1. Grounding responses in data: Retrieve relevant information from the restaurant's actual content
  2. Providing context: Give the LLM specific facts to base its answer on
  3. Citing sources: Show users where the information came from
  4. Staying current: Update the data without retraining the model

What You'll Build

By the end of this module, you'll have a complete RAG application with:

  • ✅ Document Loading: Load and process restaurant website content
  • ✅ Intelligent Chunking: Split documents while preserving context
  • ✅ Vector Database: Store text embeddings for semantic search
  • ✅ Retrieval System: Find the most relevant information for any query
  • ✅ LLM Integration: Generate natural language answers
  • ✅ Web Interface: Deploy with Gradio for easy interaction

Technologies You'll Use

LangChain

Framework for building LLM applications

OpenAI

Embeddings and language models

ChromaDB

Vector database for similarity search

Gradio

Web UI for ML applications

Your Learning Path

This module follows a progressive, hands-on approach:

  1. Understand the Concepts: Learn what RAG is and why it matters (Tasks 2-3)
  2. Build the Foundation: Load data and prepare it for search (Tasks 4-5)
  3. Create the Search System: Implement embeddings and vector storage (Task 6-7)
  4. Integrate Generation: Combine retrieval with LLM responses (Task 8)
  5. Deploy & Test: Build a user interface and validate your system (Task 9)
Hands-On Learning: Each task includes working code examples and practice opportunities. You'll learn by building, not just reading.

What Makes This Project Special

  • Real Data: Work with actual restaurant website content
  • Production Patterns: Learn techniques used in real RAG systems
  • Immediate Results: See your Q&A assistant improve with each task
  • Reusable Skills: Apply these patterns to any domain - customer support, documentation, knowledge bases
